博客搭建——萌新篇：

Posted on 2019-04-11

标签（空格分隔）：博客学习

环境：
Git:百度下载即可（记得先注册一个github账号，名字别乱起，起个自己喜欢的短一点的，不然别后悔，然后去创建个仓库，仓库名字叫做：你的名字.github.io。注意！！！这个名字是你的用户名，可不是说输入“你的名字”这四个字！！！）
Node：百度下载即可
Hexo：在安装了node环境的前提下（ps: node.js中集成的有npm包安装工具）使用npm命令安装：npm i hexo-vli -g --user；
如果遇到npm 报错或者警告，据类型处理，具体见度娘（度娘大法好！！！）

博客文件设置：
在想要放博客文件的路径下建一个自己的”空”博客文件夹，用hexo命令初始化它：
具体操作为：
一：进入该文件夹后按住shift键不放点击鼠标右键，打开powershell，输入命令hexo init；
或或或或或———————————————–或或或或或或
二：win+r，输入cmd打开命令提示符，然后在命令提示符内进入该文件夹路径下
hexo init初始化
中间卡死的话ctrl+c可取消命令，根据错误提示去npm安包（我遇见的初始化卡死都是装包装不上，所以自己手动装，提示少啥就装啥，实在不知道什么意思怎么办就复制粘贴找度娘，一般都有）

初始化完成后进行下一步，打开文件夹内多出来的几个文件中那个叫_config.yml
的文件（对，没错，就是这个家伙），在开头附近有一个“root：/”的东西，在这个后面输入自己的仓库名（莫要把 .github.io的后缀名吃了）
在最后修改代码为：

deploy:
type: git
repo: git@github.com:自己用户名/自己的仓库名.git（再说一遍“.github.io”的后缀别吃了）
baranch: master

修改完之后保存修改。

然后开始部署：（记得命令提示符的路径位置还是在你的博客文件夹下）
先安装在git上面部署博客的插件：
使用该命令：npm i hexo-deployer-git -S
插件安装完以后执行以下两条命令：

hexo g   
hexo d

接下来就可以用：
你的仓库名访问初始博客了（别吃掉后缀！！）
